# 电赛2026 - 智能循迹小车项目集

## 项目概述

本项目集包含四个独立的子项目，构成了一个完整的智能循迹小车系统，专门为2026年电子设计竞赛（电赛2026）设计。系统以STM32F103C8T6为主控制器，实现黑线循迹、8字路线轨迹、路程点停车等高级功能，并通过ESP32进行蓝牙无线数据传输。

**核心特性**:
- **模块化设计**: 三个STM32项目分别对应不同竞赛题目，共享底层硬件驱动
- **实时控制**: 10ms中断周期PID控制，确保循迹稳定性
- **状态机管理**: 8字路线采用状态机精确控制各阶段切换
- **路程点停车**: 支持多点停车等待和自动恢复功能
- **无线通信**: ESP32实现蓝牙SPP串口透传，实时传输调试数据

## 项目列表

### 1. 第一题：基础黑线循迹 (`04_06_17_17工程备份_第一题`)
**功能**: 8路灰度传感器黑线循迹，PID控制，路程累计，路程点触发
- **核心算法**: 加权误差PID循迹控制
- **距离跟踪**: 编码器脉冲距离累计，可配置停止阈值
- **人机交互**: OLED显示，蜂鸣器提示，按键模式选择
- **通信接口**: UART调试输出，预留视觉/语音模块接口

### 2. 第二题：8字路线状态机 (`04_09_12_13工程备份_第二题_稍稳定`)
**功能**: 8字路线轨迹状态机控制，几何轨迹规划，角度对齐
- **状态机设计**: 8阶段状态机管理A→C→B→D→A闭合轨迹
- **几何参数**: 对角线102cm，弧线83cm，旋转角度50°
- **触发机制**: 距离触发 + 传感器触发双保险
- **角度对齐**: MPU6050航向角对齐，确保轨迹闭合精度

### 3. 第三题：路程点停车模块 (`04_09_14_07工程备份_第三题运动模块`)
**功能**: 在基础循迹上增加路程点停车等待和自动恢复功能
- **停车管理**: 到达预设路程点自动停车，可配置等待时间（默认5秒）
- **速度保存**: 停车前保存当前速度，恢复时保持连续性
- **自动恢复**: 等待结束后自动恢复原速度行驶
- **多重提示**: 蜂鸣器、LED、串口输出三重状态提示

### 4. ESP32蓝牙串口服务器 (`spp_server`)
**功能**: 蓝牙BLE SPP串口透传，无线转发STM32调试数据
- **无线传输**: 将STM32串口输出通过BLE转发到手机/电脑
- **引脚优化**: 使用UART1避免占用USB串口（GPIO9-RX, GPIO10-TX）
- **兼容性**: 支持Android/iOS蓝牙串口APP
- **数据分包**: 自动处理长数据包分包传输

## 硬件架构

### 主控芯片
- **STM32F103C8T6**: ARM Cortex-M3内核，72MHz主频，64KB Flash，20KB RAM
- **ESP32-WROOM-32**: 双核处理器，集成蓝牙BLE 4.2，用于无线通信

### 传感器模块
1. **8路灰度传感器阵列**
   - 检测方式：数字输出（1=黑线，0=背景）
   - 布局：D1~D8从左到右等间距排列
   - 多路切换：3位地址线选择8个通道

2. **MPU6050六轴传感器**
   - 三轴加速度计 + 三轴陀螺仪
   - I2C接口通信
   - 提供航向角(yaw)用于角度环PID控制

3. **增量式编码器** ×2
   - 左右电机各一个，AB相输出
   - 提供速度反馈用于速度环PID控制

### 执行机构
1. **直流减速电机** ×2
   - 左右轮独立驱动
   - TB6612FNG双H桥电机驱动
   - PWM调速控制

2. **电机驱动电路**
   - TB6612FNG驱动芯片
   - 最大电流1.2A，支持正反转控制

### 人机交互
1. **OLED显示屏** (0.96寸，128×64)
   - I2C接口，实时显示系统状态
2. **蜂鸣器**
   - 路程点触发、停车、恢复等事件提示
3. **LED指示灯**
   - 系统运行状态指示
4. **按键** ×3
   - PB12、PB13、PB14，用于模式选择和功能控制

### 通信接口
1. **UART串口** ×3
   - UART1: 调试输出(115200bps) → ESP32蓝牙转发
   - UART2: 视觉识别模块通信（预留）
   - UART3: 语音模块通信（预留）
2. **I2C总线**
   - PB10(SCL)、PB11(SDA)，连接MPU6050和OLED
3. **蓝牙BLE**
   - ESP32实现蓝牙SPP串口透传服务

## ESP32与STM32连接

### 物理连接
```
STM32 (115200波特率)      ESP32 (UART1)
-------------------      --------------------
PA9 (TX)  ----------->  GPIO9   (UART1_RX)
PA10 (RX) <-----------  GPIO10  (UART1_TX) - 可选，本项目单向传输
GND       ------------  GND
```

### 蓝牙配置
- **设备名称**: `nimble-ble-spp-svr` (可在ESP32代码中修改)
- **服务UUID**: 标准SPP服务 (0xABF0-0xABF4)
- **波特率**: 115200, 8N1 (与STM32 `printf`输出匹配)

### 手机端测试
1. **Android**: 安装"Serial Bluetooth Terminal" APP
2. **iOS**: 安装"LightBlue"或其他BLE调试工具
3. 搜索并连接`nimble-ble-spp-svr`设备
4. STM32发送的`printf()`数据将实时显示在手机端

## 编译与使用

### STM32项目编译
所有STM32项目使用Keil MDK-ARM (µVision)开发环境：

1. **打开项目文件**: `user/Project.uvprojx`
2. **选择目标设备**: STM32F103C8
3. **编译生成HEX**: F7或Project → Build Target
4. **烧录到STM32**: 使用ST-Link或J-Link调试器
5. **输出文件**: 生成在`user/Objects/`目录下

### ESP32项目编译
ESP32项目使用ESP-IDF开发框架：

```bash
# 进入项目目录
cd "spp_server"

# 配置项目（可选）
idf.py menuconfig

# 编译项目
idf.py build

# 烧录到ESP32
idf.py -p COMx flash monitor  # COMx为ESP32的串口号
```

### 系统启动流程
1. **硬件连接**: 按接线表连接所有模块
2. **上电顺序**: 先上STM32，再上电机电源
3. **系统初始化**: 蜂鸣器响一声，LED闪烁一次
4. **自动运行**: 小车自动开始循迹任务
5. **状态提示**: 路程点触发、停车、恢复等事件有蜂鸣器提示
6. **无线监控**: 手机连接ESP32蓝牙，实时查看串口输出

│   ├── ml_pwm.c/.h     # PWM输出
│   ├── ml_tim.c/.h     # 定时器配置
│   └── ml_uart.c/.h    # 串口通信

### 关键文件说明
- **main.c**: 项目主程序，包含初始化、参数配置、主循环逻辑
- **isr.c**: 中断服务程序，处理TIM3(10ms控制周期)、EXTI(编码器/MPU6050中断)
- **pid.c**: PID控制器实现，包含位置式和增量式两种算法
- **gray_track.c**: 8路灰度传感器读取和循迹控制算法
- **motor.c**: 电机PWM驱动和编码器接口

## 调试与监控

### 串口调试输出
- **波特率**: 115200
- **数据内容**: 
  - 航向角数据（每500ms输出）
  - 路程点触发状态
  - 停车/恢复事件
  - 状态机切换信息（第二题）
  - 系统错误信息

### OLED显示内容
- 当前运行模式
- 累计路程（cm）
- 当前速度
- 航向角度
- 路程点状态

### 状态提示
- **蜂鸣器**: 短响一声 = 路程点触发，长响 = 停车/恢复
- **LED**: 闪烁 = 系统启动，常亮/灭 = 运行状态
- **串口输出**: 详细状态信息，可通过ESP32蓝牙无线查看

### 调试建议
1. **分模块测试**: 先测试电机单独运行，再测试传感器，最后集成
2. **参数渐进调整**: 从小数值开始逐步调整PID参数
3. **实时监控**: 使用串口输出实时监控控制变量
4. **硬件检查**: 定期检查接线是否松动，电源是否稳定
